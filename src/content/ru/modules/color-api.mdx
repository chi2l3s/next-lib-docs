# Color API

Color API в NextLib предельно простой: один интерфейс и одна основная операция форматирования.

## Интерфейс

```java
public interface ColorUtil {
    String formatMessage(String message);
}
```

## `formatMessage(String message)`

Назначение: преобразовать строку с цвет-кодами в формат, который Minecraft-клиент корректно отобразит.

Поддерживаемые входы:

- `&`-коды (`&a`, `&7`, `&c`);
- HEX запись вида `&#RRGGBB`.

Поведение:

- если `message == null`, возвращается пустая строка;
- если цветов нет, возвращается исходный текст в обработанном виде.

## Пример

```java
String text = NextLib.c.formatMessage("&aПривет, &#3498dbигрок");
player.sendMessage(text);
```

## Где использовать

- Сообщения игрокам.
- Заголовки GUI.
- Названия и лор предметов.

## Типичная ошибка

Цвета не применяются, если сообщение отправляется в обход форматтера. Для всех пользовательских строк лучше использовать единый вызов `formatMessage`.
